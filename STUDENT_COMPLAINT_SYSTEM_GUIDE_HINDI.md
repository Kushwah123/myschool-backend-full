# 🎓 स्कूल मैनेजमेंट सिस्टम - स्टूडेंट कंप्लेंट सिस्टम गाइड

## 📋 अवलोकन (Overview)
यह गाइड आपको बताएगा कि स्कूल में स्टूडेंट के खिलाफ कंप्लेंट कैसे रेजिस्टर की जाती है। यह सिस्टम पूरी तरह ऑटोमेटेड है और WhatsApp के जरिए स्टूडेंट या उसके पैरेंट को नोटिफिकेशन भेजता है।

---

## 👨‍🏫 1. टीचर के लिए कंप्लेंट रेज करना (Teacher: Raise Student Complaint)

### 📍 कहाँ मिलेगा: Teacher Panel → Complaint Form

### 🎯 उद्देश्य:
टीचर स्टूडेंट के खिलाफ कंप्लेंट रेज करता है और ऑटोमेटिकली WhatsApp पर नोटिफिकेशन भेजता है।

### 📝 कैसे इस्तेमाल करें:

#### **Step 1: क्लास चुनें**
```
Class: Class 6 A
```
- सबसे पहले वह क्लास चुनें जिसमें स्टूडेंट है

#### **Step 2: स्टूडेंट चुनें**
```
Student: राहुल कुमार
```
- सेलेक्टेड क्लास के सभी स्टूडेंट्स की लिस्ट दिखेगी

#### **Step 3: सब्जेक्ट टेम्प्लेट चुनें या कस्टम सब्जेक्ट लिखें**
**प्री-डिफाइंड टेम्प्लेट्स:**
- Behavior issue (व्यवहार संबंधी समस्या)
- Attendance concern (उपस्थिति संबंधी चिंता)
- Homework not submitted (होमवर्क न करना)
- Late arrival (देरी से आना)
- Academic performance (अकादमिक प्रदर्शन)
- Uniform violation (यूनिफॉर्म उल्लंघन)
- Other (अन्य)

**या कस्टम सब्जेक्ट:**
```
Subject: Fighting in class
```

#### **Step 4: डिटेल डिस्क्रिप्शन लिखें**
```
Description:
राहुल आज क्लास में लड़ाई कर रहा था। उसने अपने क्लासमेट से मारपीट की और टीचर की बात नहीं मानी। ऐसा व्यवहार स्कूल के नियमों के खिलाफ है।
```

#### **Step 5: Create Complaint बटन पर क्लिक करें**

### ✅ परिणाम:
- कंप्लेंट डेटाबेस में सेव हो जाती है
- स्टूडेंट या उसके पैरेंट को WhatsApp मैसेज जाता है
- कंप्लेंट का स्टेटस "Open" सेट होता है

## 📱 2. WhatsApp मैसेज का फॉर्मेट (WhatsApp Message Format)

जब कंप्लेंट रेज की जाती है तो रेसिपिएंट को यह मैसेज मिलता है:

```
*New Complaint Raised*

Student: राहुल कुमार
Teacher: Mr. Sharma (Mathematics Teacher)
Subject: Fighting in class
Description: राहुल आज क्लास में लड़ाई कर रहा था। उसने अपने क्लासमेट से मारपीट की और टीचर की बात नहीं मानी। ऐसा व्यवहार स्कूल के नियमों के खिलाफ है।

Please respond to the teacher if needed.
```

## 📊 3. कंप्लेंट का स्टेटस ट्रैकिंग (Complaint Status Tracking)

### 📍 स्टेटस टाइप्स:

#### **🟢 Open (खुला)**
- नई कंप्लेंट रेज हुई है
- अभी कोई एक्शन नहीं लिया गया

#### **🟡 In Progress (प्रगति में)**
- एडमिन या टीचर ने कंप्लेंट पर काम शुरू किया
- जांच चल रही है

#### **🔴 Resolved (हल हो गया)**
- कंप्लेंट का समाधान हो गया
- मामले को बंद कर दिया गया

---

## 🔍 4. कंप्लेंट डिलिवरी सिस्टम (Complaint Delivery System)

### 📱 प्राथमिकता क्रम:
1. **पहला प्रयास:** स्टूडेंट का खुद का मोबाइल नंबर
2. **सेकंड प्रयास:** पैरेंट का मोबाइल नंबर (अगर स्टूडेंट का नंबर नहीं है)

### ✅ सक्सेस कंडीशन:
- वैलिड मोबाइल नंबर होना चाहिए
- WhatsApp सर्विस एक्टिव होनी चाहिए
- मैसेज डिलिवर होना चाहिए

### ❌ फेलियर के कारण:
- स्टूडेंट/पैरेंट का मोबाइल नंबर रजिस्टर्ड नहीं
- नंबर इंवैलिड या फॉर्मेट गलत
- WhatsApp सर्विस डाउन
- इंटरनेट कनेक्शन प्रॉब्लम

## 👨‍💼 5. एडमिन के लिए कंप्लेंट मैनेजमेंट (Admin: Complaint Management)

### 📍 कहाँ मिलेगा: Admin Panel → Complaints

### 🎯 एडमिन की रोल:
- सभी कंप्लेंट्स देखना
- स्टेटस अपडेट करना
- टीचर्स को गाइड करना
- रिपोर्ट्स जनरेट करना

```
📊 Complaint Statistics:
🟢 Open: 5
🟡 In Progress: 2
🔴 Resolved: 15
```

### 📋 कंप्लेंट टेबल:
| Student | Teacher | Subject | Status | Created | Actions |
|---------|---------|---------|--------|---------|---------|
| राहुल कुमार | Mr. Sharma | Fighting | 🟢 Open | 15/1/24 | 📄 View 🔄 Update |

### 🎯 एक्शन्स:
- **📄 View:** कंप्लेंट की फुल डिटेल्स देखें
- **🔄 Update:** स्टेटस चेंज करें (Open → In Progress → Resolved)

## 📋 6. पूरा कंप्लेंट वर्कफ्लो (Complete Complaint Workflow)

### 🎯 स्कूल में कंप्लेंट प्रोसेस:

```
1. टीचर कंप्लेंट रेज करता है
   ├── क्लास और स्टूडेंट सेलेक्ट करता है
   ├── सब्जेक्ट और डिस्क्रिप्शन लिखता है
   ├── Create Complaint बटन प्रेस करता है

2. सिस्टम ऑटोमेटिकली:
   ├── डेटाबेस में कंप्लेंट सेव करता है
   ├── स्टूडेंट का मोबाइल नंबर ढूंढता है
   ├── अगर नहीं मिला तो पैरेंट का नंबर लेता है
   ├── WhatsApp मैसेज तैयार करता है
   └── रेसिपिएंट को मैसेज भेजता है

3. स्टेटस ट्रैकिंग:
   ├── शुरू में: Open
   ├── एडमिन जांच करता है: In Progress
   ├── समाधान होता है: Resolved

4. WhatsApp डिलिवरी:
   ├── सक्सेस: कंप्लेंट रिकॉर्ड में नोट होता है
   ├── फेल: एडमिन को नोटिफाई किया जाता है
   └── मैनुअल फॉलोअप किया जाता है
```

## 🔧 7. बैकएंड आर्किटेक्चर (Backend Architecture)

### 📊 डेटाबेस स्कीमा (Complaint Model):
```javascript
{
  studentId: ObjectId (ref: 'Student'),
  teacherId: ObjectId (ref: 'User'),
  subject: String (required),
  description: String (required),
  status: Enum ['Open', 'In Progress', 'Resolved'],
  whatsappSent: Boolean,
  whatsappResult: Mixed,
  createdAt: Date,
  updatedAt: Date
}
```

### 🛠️ API एंडपॉइंट्स:

#### **POST /complaints**
- कंप्लेंट क्रिएट करता है
- WhatsApp नोटिफिकेशन भेजता है
- सिर्फ टीचर एक्सेस

#### **GET /complaints**
- सभी कंप्लेंट्स लिस्ट करता है
- सिर्फ एडमिन एक्सेस

#### **GET /complaints/student/:studentId**
- स्पेसिफिक स्टूडेंट की कंप्लेंट्स
- एडमिन एक्सेस

#### **GET /complaints/teacher/:teacherId**
- स्पेसिफिक टीचर की कंप्लेंट्स
- टीचर और एडमिन एक्सेस

## 📱 8. WhatsApp इंटीग्रेशन कैसे काम करता है (How WhatsApp Integration Works)

### 🔧 मैसेज तैयार करने का प्रोसेस:

1. **कंप्लेंट डेटा कलेक्ट करना:**
   ```javascript
   const complaintData = {
     studentName: student.fullName,
     teacherName: teacher.name,
     subject: complaint.subject,
     description: complaint.description
   }
   ```

2. **मैसेज फॉर्मेट बनाना:**
   ```javascript
   const message = `*New Complaint Raised*\n\nStudent: ${studentName}\nTeacher: ${teacherName}\nSubject: ${subject}\nDescription: ${description}\n\nPlease respond to the teacher if needed.`
   ```

3. **फोन नंबर फाइंड करना:**
   ```javascript
   // पहला: स्टूडेंट का नंबर
   let phoneNumber = student.mobile

   // अगर नहीं तो पैरेंट का नंबर
   if (!phoneNumber) {
     const parent = await Parent.findOne({ studentIds: student._id })
     phoneNumber = parent.mobile
   }
   ```

4. **WhatsApp सर्विस से भेजना:**
   ```javascript
   const result = await whatsappService.sendMessage(phoneNumber, message)
   ```

5. **रिजल्ट सेव करना:**
   ```javascript
   complaint.whatsappSent = result.success
   complaint.whatsappResult = result
   ```

## ⚠️ 9. महत्वपूर्ण नियम और कैव्हेट्स (Important Rules & Caveats)

### 📝 कंप्लेंट बनाने के नियम:

#### **✅ जरूरी फील्ड्स:**
- Class (क्लास सेलेक्ट करना)
- Student (स्टूडेंट सेलेक्ट करना)
- Subject (सब्जेक्ट टेम्प्लेट या कस्टम)
- Description (डिटेल डिस्क्रिप्शन)

#### **✅ ऑथोराइजेशन:**
- सिर्फ टीचर कंप्लेंट रेज कर सकता है
- एडमिन सभी कंप्लेंट्स मैनेज कर सकता है
- स्टूडेंट या पैरेंट कंप्लेंट नहीं रेज कर सकता

### 🚫 प्रतिबंध:

#### **टीचर के लिए:**
- सिर्फ अपनी क्लास के स्टूडेंट्स के लिए कंप्लेंट रेज कर सकता है
- एक ही स्टूडेंट के लिए multiple कंप्लेंट्स रेज कर सकता है
- कंप्लेंट का स्टेटस चेंज नहीं कर सकता (सिर्फ एडमिन)

#### **एडमिन के लिए:**
- सभी कंप्लेंट्स देख सकता है
- स्टेटस अपडेट कर सकता है
- कंप्लेंट डिलीट नहीं कर सकता (सिर्फ अपडेट)

### 📊 WhatsApp डिलिवरी:

#### **डिलिवरी प्राथमिकता:**
```
1. स्टूडेंट का मोबाइल नंबर
2. पैरेंट का मोबाइल नंबर
3. अगर दोनों नहीं तो कंप्लेंट रिकॉर्ड में नोट
```

#### **सक्सेस रेट कैल्कुलेशन:**
```
Total Complaints = सभी कंप्लेंट्स
WhatsApp Sent = कितने को मैसेज गया
WhatsApp Failed = कितने को मैसेज नहीं गया
```

## 🆘 10. कॉमन प्रॉब्लम्स और समाधान (Common Problems & Solutions)

### ❌ **प्रॉब्लम: कंप्लेंट सबमिट नहीं हो रही**
**समाधान:**
- चेक करें कि सभी फील्ड्स भरें हैं
- चेक करें कि क्लास और स्टूडेंट सेलेक्टेड हैं
- चेक करें कि आप लॉगिन हैं

### ❌ **प्रॉब्लम: WhatsApp मैसेज नहीं गया**
**समाधान:**
- चेक करें कि स्टूडेंट/पैरेंट का मोबाइल नंबर रजिस्टर्ड है
- चेक करें कि WhatsApp सर्विस रेडी है
- एडमिन से संपर्क करें मैनुअल नोटिफिकेशन के लिए

### ❌ **प्रॉब्लम: स्टूडेंट लिस्ट नहीं दिख रही**
**समाधान:**
- चेक करें कि क्लास सेलेक्ट की है
- चेक करें कि क्लास में स्टूडेंट्स हैं
- पेज रिफ्रेश करें

### ❌ **प्रॉब्लम: सब्जेक्ट टेम्प्लेट काम नहीं कर रहा**
**समाधान:**
- "Other" सेलेक्ट करें कस्टम सब्जेक्ट के लिए
- या प्री-डिफाइंड टेम्प्लेट इस्तेमाल करें

## 🎯 12. बेस्ट प्रैक्टिसेस (Best Practices)

### 👨‍🏫 टीचर के लिए:
- **क्लियर सब्जेक्ट लिखें:** "Fighting in class" न कि "Bad behavior"
- **डिटेल डिस्क्रिप्शन दें:** क्या हुआ, कब हुआ, कौन शामिल था
- **टेम्प्लेट इस्तेमाल करें:** जहां संभव हो प्री-डिफाइंड सब्जेक्ट चुनें
- **तथ्य आधारित रखें:** व्यक्तिगत राय से ज्यादा ऑब्जर्वेशन लिखें

### 👨‍💼 एडमिन के लिए:
- **रेगुलर मॉनिटर करें:** नए कंप्लेंट्स चेक करें
- **टाइमली रिस्पॉन्स दें:** स्टेटस अपडेट करें
- **फॉलोअप करें:** रिजॉल्व्ड कंप्लेंट्स ट्रैक करें
- **रिपोर्ट्स बनाएं:** कंप्लेंट ट्रेंड्स एनालाइज करें

### 📱 WhatsApp के लिए:
- **वैलिड नंबर अपडेट रखें:** स्टूडेंट और पैरेंट के नंबर
- **सर्विस मॉनिटर करें:** WhatsApp कनेक्शन चेक रखें
- **बैकअप प्लान:** अगर WhatsApp फेल हो तो SMS या कॉल

## 📋 13. कंप्लेंट कैटेगरी गाइड (Complaint Category Guide)

### 🎯 कॉमन कंप्लेंट टाइप्स:

#### **📚 अकादमिक (Academic):**
- Homework not submitted (होमवर्क न करना)
- Poor performance (खराब प्रदर्शन)
- Not paying attention (ध्यान न देना)

#### **👥 व्यवहारिक (Behavioral):**
- Fighting in class (क्लास में लड़ाई)
- Bullying (बुलिंग)
- Disrespect to teacher (टीचर का सम्मान न करना)

#### **⏰ उपस्थिति संबंधी (Attendance):**
- Late arrival (देरी से आना)
- Early departure (जल्दी जाना)
- Absent without notice (बिना बताए गायब)

#### **👔 अनुशासन संबंधी (Disciplinary):**
- Uniform violation (यूनिफॉर्म उल्लंघन)
- Mobile phone usage (मोबाइल इस्तेमाल)
- Breaking school rules (स्कूल नियम तोड़ना)
